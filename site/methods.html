Methods

Methods

Where the data comes from, the rule that defines AI, the metrics, and what it can and cannot carry.

Principle

Open data, open rule, every choice on the page.

Every input is a public record. No licensed database is used. The rule that decides what counts as AI is written in one file, so anyone can read it, change it, and rerun the study.

Data sources

Two public feeds

USAspending contract awards, pulled through the public API. The OMB 2025 Federal AI Use Case Inventory.

Sample

2,107 awards, FY2016 to 2025

Contract award types only. Of 2,107 awards, 1,947 met the rule in window. The inventory holds 3,611 use cases, 1,023 acquired by contract.

The rule

What counts as an AI contract

An award counts as AI if its description contains one of these terms. Word-boundary checks stop AI from matching email, Hawaii, or a building code like AI-3.

artificial intelligence · machine learning · deep learning · neural network · natural language processing · computer vision · large language model · foundation model · generative AI · AI · NLP · LLM

Cross-check

Matched to the inventory

Keyword awards are compared with the inventory at the agency level, using the contracted subset.

Reproducibility

Three commands

pip install -r requirements.txt python fetch_usaspending.py python analyze.py

Coverage and limits

What the data cannot carry

Use cases are not dollars, so the cross-check measures coverage, not a corrected total. Award Amount is an obligation proxy, not a final outlay. Both sources are biased in opposite directions: the keyword method toward Defense, the inventory toward civilian agencies, because Defense does not report. Keyword search reads base award descriptions, so AI bought inside task orders goes unseen. The broad-definition test is partial and needs a pull by industry code.
