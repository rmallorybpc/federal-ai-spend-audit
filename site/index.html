The count depends on the words you pick

I ran the same data twice. The only change was the list of AI terms.

Two phrases returned 1,600 contract awards. Nine phrases returned 1,979. Nothing about federal AI changed between those runs. Only the definition did.

Every headline figure you have read rests on a word list like this one. Most do not show you the list.

The top AI vendor is a measurement artifact

MathWorks is the largest AI contractor by the keyword method. MathWorks makes MATLAB.

Those awards are very likely tool and license purchases that match because "machine learning" sits in the product description. A tool license is not an AI program. The keyword method cannot tell the difference. This is what a loose definition looks like in practice.

The rule, stated in the open

An award counts as AI if its description contains one of these terms. Word-boundary checks stop "AI" from matching "email," "Hawaii," or a building code like "AI-3."

artificial intelligence · machine learning · deep learning · neural network · natural language processing · computer vision · large language model · foundation model · generative AI · AI · NLP · LLM

The cross-check uses the OMB inventory, filtered to the use cases agencies acquired by contract. That subset is the fair comparison to contract spend. The inventory holds 3,611 reported use cases across 41 agencies. 1,023 were acquired by contract.

What I am not claiming

The limits matter as much as the findings.

  • Use cases are not dollars.The inventory counts systems, not obligations. This measures a gap in coverage, not a corrected total.
  • Both sources are biased, in opposite directions.The keyword method leans toward Defense. The inventory leans civilian, because Defense does not report. Neither is the truth on its own.
  • Keyword search reads base award descriptions.AI bought inside task orders and modifications goes unseen. My count runs lower than proprietary datasets, which fits that gap.
  • The broad-definition test is partial.A full version needs a pull by industry code, not by keyword. That is the next build, and it is marked unfinished.
